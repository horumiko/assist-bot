# 1\. Обзор проекта

Персональный Telegram-бот на базе LLM для управления задачами и календарём. Бот проактивно напоминает о задачах, подсказывает обновить статусы и синхронизирует изменения между Todoist и Bitrix24. Трекает все активные задачи независимо от того, кто их создал.

| **Параметр** | **Значение** |
| --- | --- |
| Тип | Персональный (single-user) |
| Язык | Русский |
| Стек | TypeScript / Node.js |
| LLM | `LLM_MODEL` + `LLM_ANALYSIS_MODEL` (роутинг по сценариям) |
| Хостинг | Свой VPS |
| Интеграции | Todoist + Google Calendar + Bitrix24 (REST webhook) |

# 2\. Система меток в Todoist

В Todoist используются две служебные метки, которые управляют поведением бота:

| **Метка** | **Назначение** |
| --- | --- |
| bot | Ставится автоматически на все задачи, созданные через бота. Позволяет отличать их от задач, созданных вручную. |
| bitrix | Ставится на задачи, связанные с корпоративным Bitrix24. При обновлении статуса/дедлайна бот дублирует изменения в Bitrix24. В описании задачи хранится ссылка на Bitrix24-задачу. |

## 2.1 Правила меток

- Метка bot ставится автоматически при создании задачи через бота. Пользователь не должен указывать её вручную.
- Метка bitrix может быть поставлена вручную (пользователь сам отмечает в Todoist) или через бота («привяжи задачу X к битриксу TASK_ID»).
- Задача может иметь обе метки одновременно (создана ботом + синк с Bitrix24).
- Задача может не иметь ни одной метки - это обычная ручная задача без связи с Bitrix24.

## 2.2 Формат связи с Bitrix24

Задачи с меткой bitrix должны содержать ссылку на Bitrix24-задачу в поле description. Бот распознаёт два формата:

- \[bitrix:12345\] - короткий формат (ID задачи)
- <https://company.bitrix24.ru/workgroups/group/1/tasks/task/view/12345/> - полная URL

Бот парсит ID задачи из описания и использует его для API-вызовов.

## 2.3 Скоуп трекинга

Бот трекает ВСЕ активные задачи в Todoist для брифингов и отчётов - независимо от наличия меток. Метки влияют только на поведение при обновлении:

- Задача без метки bitrix → обновляется только в Todoist
- Задача с меткой bitrix → обновляется в Todoist + дублируется в Bitrix24 (статус, комментарий, дедлайн)

# 3\. Ключевые сценарии

## 3.1 Ежедневный утренний брифинг

**Когда:** Каждый день в 08:35 (Europe/Minsk)

**Скоуп:** Все активные задачи Todoist (на всех проектах) + события Google Calendar

Содержание:

- Все задачи со статусом «в работе»
- Задачи требующие обновления статуса (>48ч) - через меню бота (кнопка «📝 Дать статус»)
- События на сегодня из Google Calendar
- Просроченные задачи

### Механика трекинга статусов

Таблица task_status_log в supabase: todoist_task_id, bitrix_task_id (nullable), last_status_update, last_status. Если >48ч - задача попадает в блок «нужно обновить».

Способы обновления:

- Через меню «📝 Дать статус» и выбор задачи/колонки reply-кнопками
- Написать апдейт текстом → бот обновляет Todoist + Bitrix24 (если есть метка bitrix)

### Синхронизация с Bitrix24

При обновлении задачи с меткой bitrix бот выполняет:

- Обновление метки/проекта в Todoist
- Обновление стадии в Bitrix24 Kanban (task.stages.movetask)
- Добавление комментария в Bitrix24 (task.commentitem.add)
- Обновление дедлайна в Bitrix24 (tasks.task.update DEADLINE)
- Обновление task_status_log

## 3.2 Пятничный отчёт

**Когда:** Пятница 08:35

**Скоуп:** Все активные на проектах + завершённые за неделю

**Формат:** Личный, неформальный

Содержание: завершённые, в работе, просроченные, LLM-сводка с анализом.

## 3.3 Создание задач и встреч

| **Сообщение** | **Действие** |
| --- | --- |
| «Встреча с Артёмом завтра в 15:00» | Calendar: создать событие |
| «Каждый вт в 10 standup» | Calendar: recurring event |
| «Задача: презентация, дедлайн среда» | Todoist: задача + метка bot |
| «Задача X готова» | Todoist: complete. Если метка bitrix → + Bitrix24 статус |
| «По X жду фидбек» | Todoist: метка. Если bitrix → + статус + коммент |
| «Перенеси дедлайн X на пятницу» | Todoist: due. Если bitrix → + Bitrix24 DEADLINE |
| «Привяжи X к битриксу 12345» | Todoist: +метка bitrix + \[bitrix:12345\] в description |

## 3.4 Уведомления о встречах

Напоминание за 15 мин до события: название, время, ссылка.

# 4\. Интеграции

## 4.1 Todoist

**API:** REST API v2, персональный токен

**Статусы:** Метки/проекты = статусы (маппинг при setup)

**Служебные метки:** bot, bitrix

| **Операция** | **Описание** |
| --- | --- |
| Чтение | Все активные, просроченные, completed за период |
| Создание | Задачи + авто-метка bot |
| Обновление | Статус, дедлайн, метки, description |
| Завершение | Отметка completed |

## 4.2 Google Calendar

**API:** Google Calendar API v3, OAuth 2.0 + refresh token

| **Операция** | **Описание** |
| --- | --- |
| Чтение | События на сегодня/неделю |
| Создание | Разовые и recurring |
| Редактирование | Время, название |
| Напоминания | Проверка /5 мин, уведомление за 15 мин |

## 4.3 Bitrix24

**API:** Bitrix24 REST, Inbound Webhook (Cloud). Настройка - см. отдельный гайд.

**Направление:** Одностороннее Todoist → Bitrix24 (только запись)

**Триггер:** Наличие метки bitrix на задаче + ссылка в description

| **Операция** | **REST-метод** |
| --- | --- |
| Смена статуса | task.stages.movetask / tasks.task.update STATUS |
| Комментарий | task.commentitem.add |
| Дедлайн | tasks.task.update DEADLINE |
| Чтение | tasks.task.get |
| Стадии kanban | task.stages.get |

Маппинг статусов Todoist ↔ Bitrix24 Kanban настраивается при /start.

# 6\. Архитектура

## 6.1 Компоненты

| **Компонент** | **Описание** |
| --- | --- |
| TelegramHandler | grammY: сообщения и reply keyboard user-flow |
| LLMRouter | Function calling, intent + параметры, AI-советы |
| IntentExecutor | intent → оркестрация сервисов |
| TodoistService | CRUD задач, метки bot/bitrix, статусы |
| CalendarService | CRUD событий, recurring, OAuth2 |
| BitrixService | REST: статус, коммент, дедлайн, kanban |
| BitrixLinkParser | Парсинг \[bitrix:ID\] и URL из description |
| Scheduler | node-cron: брифинг 08:35, отчёт пятница 08:35, напоминания |
| StatusTracker | supabase: task_status_log |
| ConfigManager | supabase: настройки + маппинг |

| **Категория** | **Технология** |
| --- | --- |
| Язык | TypeScript 5.x |
| Runtime | Node.js 20 LTS |
| Telegram | grammY |
| LLM | `LLM_MODEL` + `LLM_ANALYSIS_MODEL` |
| Google | googleapis |
| Todoist | @doist/todoist-api-typescript |
| Bitrix24 | axios/fetch + Inbound Webhook |
| Планировщик | node-cron |
